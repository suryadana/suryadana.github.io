BFS Face Search: Finding People on Instagram Social Graph

01 Jul 2026 • osint • face-recognition • bfs • instagram • python • automation

Concept

Face search = BFS (Breadth-First Search) on Instagram’s social graph. Each node is an Instagram account, each edge is a follow/follower relationship.

Algorithm

flowchart TD
    A["Start: ref.jpg + target"] --> B[Scrape followers/following]
    B --> C[Download all profile pics]
    C --> D[Compare each photo with ref.jpg]
    D --> E{Similarity >= 35?}
    E -->|Yes| F[MATCH Log username]
    E -->|No| G{Any unchecked accounts?}
    G -->|Yes| H[Take next account from queue]
    H --> I[Scrape their followers/following]
    I --> C
    G -->|No| J[Done no match found]

Why BFS?

Instagram has no “search by face” feature. BFS lets us traverse the social graph without API search restrictions:

  • No API search needed — just access public profile pages
  • Gradual coverage — start from inner circle, expand outward
  • No artificial cap — only skip checked users (checked_users set)

Implementation

Queue + Checked Set

queue = deque([(target_username, 0)])   # (username, depth)
checked_users = set()
match = None

while queue and match is None:
    username, depth = queue.popleft()
    if username in checked_users:
        continue
    checked_users.add(username)

    followers = scrape_followers(username, cookie)
    for user in followers:
        sim = compare_face(ref_embedding, user["profile_pic"])
        if sim >= THRESHOLD:
            match = (user["username"], sim, depth)
            break
        if user["username"] not in checked_users:
            queue.append((user["username"], depth + 1))

Parallel Processing

Face comparison is IO-bound (downloading photos) + CPU-bound (inference). Multi-worker helps:

with ThreadPoolExecutor(max_workers=WORKERS) as pool:
    futures = []
    for user in batch:
        futures.append(pool.submit(compare_face, ref, user))

    for future in as_completed(futures):
        sim = future.result()
        if sim >= THRESHOLD:
            match = user["username"]
            pool.shutdown(wait=False)
            break

Insight

  • Negative similarity = embedding vectors point opposite directions (different person)
  • Most accounts (312/704 = 44%) have similarity < 5% — expected since profile pics vary wildly
  • Threshold 35% is tight enough to minimize false positives
  • Depth 0 alone takes ~4 min — depth 1+ can take hours or days depending on branching factor

Limitations

Issue Impact
Low-res profile pics Similarity drops ~3-7% vs original file
Private accounts Cannot scrape followers/following
Page view rate limit Still exists but far milder than API
Cookie expires ~24h Must refresh cookie manually
