Instagram Scraper: Scrape Followers/Following with Playwright

01 Jul 2026 • osint • scraping • instagram • playwright • python • automation

Problem

Instagram’s web API (web_profile_info) is very aggressive — after ~30-50 requests within minutes, the endpoint returns HTTP 429 (Too Many Requests) and stays blocked for ~15 minutes.

Solution: Browser page navigation via Playwright. With a page view, Instagram doesn’t throttle because the request looks like a regular user scrolling.

Architecture

graph TD
    A["CLI: face-osint scrape username"] --> B{Method?}
    B -->|API| C["web_profile_info (fast, risk 429)"]
    B -->|Page| D["page.goto + scroll (slow, reliable)"]
    C --> E[Parse JSON response]
    D --> F[Extract followers from DOM]
    E --> G["Save to data/username.json"]
    F --> G

Core Code

API Scraper (fast, risk 429)

def scrape_followers_api(username, cookie):
    url = f"https://www.instagram.com/api/v1/friendships/{user_id}/followers/"
    resp = requests.get(url, headers={
        "User-Agent": "Mozilla/5.0...",
        "Cookie": cookie
    })
    if resp.status_code == 429:
        raise Exception("Rate limited!")
    return resp.json()["users"]

Page Navigation Scraper (slow, anti-429)

def scrape_followers_page(username, cookie, max_count=500):
    page.set_extra_http_headers({"Cookie": cookie})
    page.goto(f"https://instagram.com/{username}/")
    page.click("a[href$='/followers/']")

    for _ in range(max_count // 12):
        page.evaluate("""
            document.querySelector('._aano')
                .scrollBy(0, 1000)
        """)
        sleep(0.3)

    items = page.query_selector_all("._aano a")
    return [el.get_attribute("href")[1:] for el in items]

Fallback Logic

def scrape_followers(username, cookie):
    try:
        return scrape_followers_api(username, cookie)
    except RateLimitError:
        log.warning(f"API 429, falling back to page view")
        return scrape_followers_page(username, cookie)

Performance Comparison

Method Speed Resilience Best For
API ~0.5s/request 429 after 30-50x Quick checks < 30 accounts
Page View ~15-30s/account Almost never rate-limited BFS depth 1+ with many accounts

Cookie

Instagram cookie must contain 3 key components:

sessionid=xxx; csrftoken=xxx; ds_user_id=xxx

Expires ~24h. Can be obtained from DevTools → Application → Cookies after login.

CLI usage:

# Direct
./face-osint --cookie "sessionid=abc; csrftoken=xyz; ds_user_id=123" scrape johndoe_official

# From file
echo "sessionid=abc; csrftoken=xyz; ds_user_id=123" > ~/ig_cookie.txt
./face-osint --cookies ~/ig_cookie.txt scrape johndoe_official

# Environment variable
export IG_COOKIE="sessionid=abc; csrftoken=xyz; ds_user_id=123"
./face-osint scrape johndoe_official

Priority: --cookie > --cookies file > IG_COOKIE env > config.py
